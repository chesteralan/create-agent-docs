## 🧱 Layered Architecture

### 1. Presentation Layer (Frontend)
- Built using **`None`**.
- State management handled by **`None`**.
- Handles client-side routing, user input capture, layout rendering, and local state sync.

### 2. Integration & Backend Layer
- Powered by a **`FastAPI`** API server.
- Responsible for executing business operations, validating input, checking security contracts, and querying the database.

### 3. Data & Storage Layer
- Powered by **`PostgreSQL`**.
- Interfaced using custom query helpers or ORMs suited for `PostgreSQL`.

---

### FastAPI Backend Architecture
- **Async Native**: All route handlers should be `async def` for non-blocking I/O.
- **Dependency Injection**: Use `Depends()` for shared logic (auth, DB sessions, pagination).
- **Auto-Validation**: Pydantic models validate request bodies, query params, and responses automatically.
- **OpenAPI Docs**: Interactive API docs at `/docs` (Swagger UI) and `/redoc` (ReDoc) are auto-generated.

---

## 🔒 Security & Data Guardrails

> [!IMPORTANT]
> - **Authentication**: Managed via `Custom`.
> - **Input Validation**: All data entering the backend boundary or Firestore database must be validated using runtime validation libraries (e.g., Zod, Valibot, or Joi) on both client and backend.
> - **Cors & Headers**: API requests must strictly adhere to the rules outlined in [API_CONTRACTS.md](API_CONTRACTS.md).
