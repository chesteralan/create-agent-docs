## 🧱 Layered Architecture

### 1. Presentation Layer (Frontend)
- Built using **`Next.js`**.
- State management handled by **`Zustand`**.
- Handles client-side routing, user input capture, layout rendering, and local state sync.

### 2. Integration & Backend Layer
- Powered by a **`None`** API server.
- Responsible for executing business operations, validating input, checking security contracts, and querying the database.

### 3. Data & Storage Layer
- Powered by **`PostgreSQL`**.
- Interfaced using custom query helpers or ORMs suited for `PostgreSQL`.

---

### Next.js App Router Architecture
- **Routing**: Uses the App Router (`app/`) with file-based routing conventions.
- **Server Components**: Default to React Server Components for data fetching and rendering.
- **Client Components**: Mark interactive components with the `'use client'` directive.
- **Layouts**: Shared layouts in `app/layout.tsx` wrap nested route segments.
- **Data Fetching**: Use `fetch` with caching directives or server-side data access patterns.

## 🔒 Security & Data Guardrails
